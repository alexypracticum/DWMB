# Плагины

Модульная система расширения [[architecture/overview|DWMB]]: 7 плагинов для различных функций.

## Список плагинов

| Плагин | Назначение | Статус |
|--------|-----------|--------|
| **AI** | Интеграция с LLM (OpenAI, Anthropic, Google) | Активен |
| **TMDB** | Импорт данных о фильмах | Активен |
| **Themes** | Управление темами оформления | Активен |
| **CMS** | Управление контентом | Активен |
| **Stats** | Статистика и аналитика | Активен |
| **RBAC** | Управление доступом | Активен |
| **Email** | Отправка уведомлений | Активен |

## Плагин как сущность

По философии [[philosophy/everything-as-entity|"Всё как сущность"]] плагины являются сущностями:

- **entity_kind** = 'plugin'
- **entity_code** — уникальный код плагина (ai, tmdb, themes, cms, stats, rbac, email)
- **projection_state** — конфигурация плагина

### Текущее состояние

Плагины реализованы как Python-модули в `app/plugins/`. Они **не являются сущностями** в БД.

**Проблема:** нет возможности управлять плагинами через интерфейс (включать/выключать/настраивать).

## Детали плагинов

### AI Plugin

См. [[plugins/ai-plugin]] для детального описания.

- Интеграция с OpenAI, Anthropic, Google
- Профили подключения
- Анализ сущностей
- Генерация описаний
- Автоматические переводы

### TMDB Plugin

Импорт данных о фильмах из TMDB API.

- Поиск фильмов
- Импорт метаданных
- Загрузка постеров
- Привязка к сущностям

### Themes Plugin

Управление темами оформления.

- Переключение тем (светлая/тёмная)
- Кастомизация CSS
- Пользовательские темы

### CMS Plugin

Управление контентом (см. [[frontend/cms]]).

- Страницы
- Меню
- Блоки контента
- Шаблоны

### Stats Plugin

Статистика и аналитика.

- Количество сущностей
- Активные пользователи
- Частота использования
- Граф связей

### RBAC Plugin

Управление доступом.

- Роли (admin, editor, viewer)
- Права доступа
- Группы пользователей

### Email Plugin

Отправка уведомлений.

- Email-уведомления
- Шаблоны писем
- SMTP-конфигурация

## Проблемы

### 1. Плагины не являются сущностями

Нет возможности управлять через интерфейс.

### 2. Нет统一的 конфигурации

Каждый плагин хранит конфигурацию по-своему.

### 3. Нет dependency injection

Нет统一的 механизма подключения плагинов.

### 4. Нет жизненного цикла

Нет管理 lifecycle плагинов (install, enable, disable, uninstall).

## Планы

- Преобразовать плагины в сущности (entity_kind = 'plugin')
- Создать CRUD для плагинов
- Внедрить统一的 конфигурацию
- Добавить dependency injection
- Управление жизненным циклом
