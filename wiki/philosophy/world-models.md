# Модели мира и проекции

**Модель мира** (ontology_model) — система отсчёта, в которой сущность имеет определённый смысл и свойства.

**Проекция** (entity_projection) — представление сущности в конкретной модели мира.

## Архитектура

```
Сущность (entity) — UUID
    ├── Проекция в модели "cinema"
    │   └── state_data: {title: "Матрица", year: 1999, rating: 8.7}
    ├── Проекция в модели "default"
    │   └── state_data: {title: "The Matrix", genre: "sci-fi"}
    └── Проекция в модели "language:ru"
        └── state_data: {label: "Матрица", description: "Фантастический фильм 1999 года"}
```

## Ключевые принципы

### Entity ≠ State

Сущность хранит только идентификатор. Данные — в projection_state как JSONB. Это обеспечивает:
- разделение идентичности и содержимого
- множество проекций для одной сущности
- версионность на уровне проекций через [[database/temporal|version_registry]]

### Связь через проекцию

[[database/relations|Семантические связи]] идут через проекции, а не напрямую через сущности. Связь реализуется через призму конкретной модели мира.

Пример: связь "режиссёр фильма" существует в модели "cinema".

### Контекст определяет истинность

Контекст (context) определяет систему отсчёта:

| Контекст | H₂O — значение |
|----------|----------------|
| Физика | Температура кипения 100°C |
| Биология | Среда для жизни |
| Экономика | Ресурс, товар |

### Достоверность данных

Поле `confidence` (NUMERIC(5,4)) определяет достоверность данных в модели мира. Позволяет:
- хранить конфликтующие данные в разных моделях
- определять приоритет источников
- взвешивать результаты запросов

## Модели мира по умолчанию

| Модель | Назначение |
|--------|-----------|
| default | Базовая модель для общих данных |
| cinema | Кинематограф (фильмы, актёры, режиссёры) |
| music | Музыка (песни, альбомы, исполнители) |
| literature | Литература (книги, авторы) |
| science | Наука (химические элементы, явления) |
| geography | География (города, страны, координаты) |
| history | История (события, периоды) |
| technology | Технологии (процессы, оборудование) |

## Соответствие философии (90%)

Реализовано:
- Entity хранит только идентификатор
- Данные разделены по проекциям
- Каждая проекция привязана к модели мира
- Поддержка нескольких проекций
- Контексты определяют систему отсчёта

Проблемы:
- ontology_model разделены на домены (art, science, social) — искусственное ограничение
- entity_kind жёстко определяет типы сущностей
