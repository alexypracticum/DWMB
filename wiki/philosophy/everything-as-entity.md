# Всё как сущность

**"Всё как сущность"** — фундаментальный принцип [[architecture/overview|DWMB]]. Любые объекты реального или абстрактного мира являются сущностями (entities) с единым идентификатором, типом, состоянием и связями.

## Ключевые положения

### 1. Сущность — это идентификатор

Сущность (entity) хранит только неизменяемый идентификатор. Все данные находятся в [[database/projections|проекциях]] — представлениях сущности в конкретной [[philosophy/world-models|модели мира]]:

- **entity** — UUID + entity_code + kind_id + status
- **projection_state** — JSONB с данными

Это обеспечивает:
- разделение идентичности и содержимого
- поддержку множественных проекций для одной сущности
- контекстную зависимость данных

### 2. Множественная природа

Одна сущность может иметь множество типов в зависимости от модели мира. Пример:

- Книга как **геометрический объект** (физическая модель)
- Книга как **источник знаний** (информационная модель)
- Книга как **объект искусства** (культурная модель)
- Книга как **топливо для печи** (утилитарная модель)

### 3. Всё должно быть сущностью

| Элемент | Тип | Статус |
|---------|-----|--------|
| Фильм, книга, песня | базовые типы | Реализовано |
| Человек | person | Реализовано |
| Язык | language | Не ENUM, а сущность |
| Поле формы | field | entity_kind = 'field' |
| Страница сайта | page | CMS как сущность |
| UI-строка перевода | ui_string | Из интерфейса |
| Плагин | plugin | Модули как сущности |
| Тип сущности | kind | Управление через UI |
| Связь | relation | Граф как сущности |
| Файл | digital_file | Медиа как сущности |

## Соответствие в текущей реализации

| Аспект | Оценка | Комментарий |
|--------|--------|-------------|
| Базовые сущности | Высокое | JSONB позволяет хранить любые данные |
| Язык как сущность | Частично | EntityKind создан, но ENUM ещё используется |
| Метки как сущности | Частично | EntityLabel существует, но не является Entity |
| Поля как сущности | Частично | FieldRegistry существует, но не является Entity |
| Страницы как сущности | **Реализовано** | page_registry мигрирован в entity kind='page' (migration 009) |
| UI-строки как сущности | Реализовано | 269 переводов мигрированы в EntityKind 'ui_string' |
| Файлы как сущности | **Компромисс** | media_asset остаётся как sidecar для производительности, CRUD через entity kind='digital_file' |
| Меню | **Удалено** | MenuItem был мёртвый код, удалён вmigration 009 |

## Проблемы и ограничения

### Предопределённые типы

Текущий список entity_kind жёстко задан в init.sql. Нельзя создать тип "автомобиль", "здание", "планета" без правки init.sql.

**Решение:** entity_kind должен быть управляемым через CRUD в админ-панели.

### field_registry

Поля жёстко привязаны к доменам (common, music, cinema и др.). По философии поля должны быть сущностями с категоризацией для фильтрации.
