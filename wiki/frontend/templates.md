# Шаблоны UI

Интерфейс [[architecture/overview|DWMB]] построен на Jinja2 + Tailwind CSS + HTMX.

## Стек

| Компонент | Технология | Назначение |
|-----------|-----------|-----------|
| Шаблонизатор | Jinja2 | Серверная генерация HTML |
| Стили | Tailwind CSS | Утилитарный CSS |
| Интерактивность | HTMX | AJAX без JavaScript |

## Структура шаблонов

```
app/templates/
├── base.html              — Базовый макет
├── entities/
│   ├── list.html          — Список сущностей
│   ├── detail.html        — Детали сущности
│   ├── create.html        — Создание сущности
│   └── edit.html          — Редактирование сущности
├── admin/
│   ├── kinds.html         — Типы сущностей
│   ├── templates.html     — Шаблоны онтологии
│   ├── fields.html        — Поля
│   └── users.html         — Пользователи
├── search/
│   └── results.html       — Результаты поиска
└── components/
    ├── nav.html           — Навигация
    ├── forms.html         — Формы
    └── pagination.html    — Пагинация
```

## Ключевые особенности

### 1. Адаптивный дизайн

Tailwind CSS обеспечивает:
- Мобильную адаптивность
- Тёмную/светлую темы
- Консистентный дизайн

### 2. HTMX для интерактивности

- Динамическая загрузка данных
- Обновление без перезагрузки страницы
- Плавная навигация

### 3. Макет страницы

```
┌─────────────────────────────────────────────┐
│  Навигация (nav.html)                       │
├──────────┬──────────────────────────────────┤
│ Sidebar  │  Основной контент                │
│          │  ┌────────────────────────────┐  │
│          │  │  Заголовок                  │  │
│          │  │  Содержимое                 │  │
│          │  │  Формы / Таблицы            │  │
│          │  └────────────────────────────┘  │
├──────────┴──────────────────────────────────┤
│  Подвал                                      │
└─────────────────────────────────────────────┘
```

## Проблемы

### 1. Нет компонентной системы

Нет переиспользуемых компонентов для:
- Форм создания/редактирования
- Таблиц данных
- Карточек сущностей

### 2. Дублирование макета

Базовый макет может дублироваться в разных шаблонах.

### 3. Ограниченная интерактивность

HTMX ограничивает сложную интерактивность (drag-and-drop, графы, визуализации).

### 4. Нет i18n в шаблонах

Переводы подставляются через `request.state.t`, но нет системы для шаблонных строк.

## Планы

- Создать библиотеку компонентов
- Оптимизировать базовый макет
- Добавить JavaScript для сложных интерактивных элементов
- Внедрить систему переводов в шаблонах
